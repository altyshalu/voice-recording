Git Термины

Workflow

Workflow — это процесс или последовательность шагов, следуемых при работе с проектом в системе контроля версий. В Git рабочий процесс может включать такие этапы, как создание веток я поняла, коммиты, слияние изменений и публикация кода в удаленном репозитории.

Rebase

Rebase — это операция в Git, которая позволяет переносить коммиты из одной ветки на другую, сохраняя линейную историю изменений. Rebase часто используется для упрощения истории и устранения лишних merge-коммитов. Операции rebase и merge в Git выполняют похожие задачи — объединяют изменения из разных веток, но они работают по-разному и имеют разные эффекты на историю коммитов.

Fetch

Fetch Fetch — это команда в Git, которая используется для получения изменений с удалённого репозитория (например, GitHub) в локальный репозиторий, без их автоматического слияния в текущую ветку. Она позволяет обновить информацию о состоянии удалённого репозитория и посмотреть, какие изменения были внесены другими разработчиками.
